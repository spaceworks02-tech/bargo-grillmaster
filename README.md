# 그릴마스터 챌린지 🔥

**경남 K-BBQ 숯불체험관광페스티벌 「불의 향연」** (2026.9.10~13, 개최지 협의 중) 홍보 및
현장 참가자용 체험 게임. (주)바르고의 **CookCam(쿡캠) 올인원숯키트** IP 연계, ZONE 3
"CookCam 칠핀체험존" 운영과 짝을 이루는 모바일 게임입니다.

> 브랜드명 표기: 실제 제품 패키지의 로고 표기는 영문 "COOKCAM"이지만, 게임 내
> 사용자 노출 텍스트(타이틀/안내문구/공유문구)는 한글 "쿡캠"으로 표기합니다.
> 코드/문서상 식별자나 파일명은 영문 CookCam을 유지합니다.

바닐라 HTML/CSS/JS로 제작된 빌드 불필요 웹 게임입니다. `suro-game` 프로젝트와 동일한
방식으로, 우선 웹 게임으로 완성한 뒤 필요 시 TWA(Android)/iOS 래퍼로 감싸 스토어에
배포하는 것을 권장합니다 (개발 기간을 크게 단축할 수 있음).

### 기획 배경 (참고 문서 기준)

`2026 K-BBQ 페스티벌` 폴더의 파일럿 계획안·스폰서십 제안서를 참고해 다음을 반영했습니다:

- 쿡캠 올인원숯키트는 **라이터만 다룰 줄 알면 누구나 쉽고 안전하게 숯불을 피울 수
  있도록** 설계된 제품입니다. 이 게임의 STAGE 2(착화)는 그 핵심 사용성(쉬운 점화)을
  재미있게 체험하도록 만든 미니게임입니다.
- 공식 키비주얼(남색 밤하늘 + 골드 캘리그라피 + 숯불 이글거림)에 맞춰 색상 팔레트를
  구성했습니다.
- 결과 화면에 바르고 OTT 드라마 「인생은 고기서 고기까지??」(2026.9월 공개) 티저 문구를
  넣어 ZONE 4(드라마 시연존)와의 교차 홍보를 반영했습니다.
- 개최지는 아직 확정되지 않았으므로(경남 10개 지자체 후보 중 협의 중) 게임 내
  특정 지역명은 넣지 않았습니다. 확정되면 타이틀 배지나 공유 문구에 추가하면 됩니다.

## 실행 방법

```bash
cd ~/kbbq-game
npx serve .
# 또는
python3 -m http.server 8080
```

브라우저에서 `http://localhost:8080` 접속. 모바일 실기기 테스트는 같은 Wi-Fi에서
`http://<PC의 IP>:8080`으로 접속하거나 ngrok 등으로 터널링하세요.

## 게임 흐름

타이틀 화면 → (게임 방법) → **STAGE 1 준비**(쿡캠 올인원숯키트 실물 패키지 언박싱 →
키트 조립 순서 맞추기, 10초) → **STAGE 2 착화**(타이밍 게이지, 3라운드)
→ **STAGE 3 굽기**(그릴 5개 포인트, 고기 반응속도 타이밍, 20초)
→ 결과·등급 판정 + 쿠폰 코드 발급 → SNS 공유 / 명예의 그릴러(로컬 리더보드) 등록

STAGE 1 진입 시 `assets/cookcam-box.png`(실제 COOKCAM BBQ SOOT 패키지 정컷)를 먼저
보여주고 "개봉하기"를 누르면 조립 미니게임이 시작됩니다.

## 목적별 활용

- **사전 홍보(SNS 바이럴)**: "결과 공유하기"를 누르면 등급/점수/쿠폰코드가 담긴
  세로형 카드 이미지를 캔버스로 즉석 생성해 `navigator.share({files:[...]})`로
  공유합니다. 이미지 공유는 iOS/Android 네이티브 공유 시트에서 **인스타그램
  스토리**가 뚜렷한 옵션으로 뜨도록 유도하는 가장 확실한 방법입니다 (텍스트만
  공유하면 인스타그램은 받을 수 있는 곳이 마땅치 않음). 브라우저가 파일 공유를
  지원하지 않으면 텍스트 전용 공유 → 클립보드 복사 순으로 자동 폴백합니다.
  ⚠️ 웹페이지에서 공유 시트 자체를 건너뛰고 인스타그램으로 강제 이동시키는 것은
  OS 정책상 불가능합니다(사용자가 항상 앱을 선택할 수 있어야 함) — 이미지 공유로
  인스타그램이 선택되기 쉽게 만드는 것이 현실적 최선입니다.
- **현장 참여 이벤트**: 부스에 태블릿/키오스크로 열어두고 방문객이 직접 플레이.
- **경품/쿠폰 연계**: 결과 화면에 6자리 쿠폰 코드(`BQ` + 랜덤 4자)를 표시.

## 밸런스 조정

`js/game.js` 상단 `CONFIG` 객체에서 조정 가능:

- `STAGE1_TIME_MS` — 준비 단계 제한시간. `STAGE1_SEQUENCE`는 실제 쿡캠 올인원숯키트
  사용설명서(6단계: 점화→숯받침→숯→타오르기→그릴이동→굽기)를 그대로 반영한 고정
  순서이므로 임의로 섞지 않음 — 순서 자체를 바꾸려면 실제 사용설명서 변경 여부부터
  확인할 것. `STAGE1_DECOYS`는 오답 유도용 무관 아이템(장갑/집게/접시).
- `STAGE2_ROUNDS`, `STAGE2_CYCLE_MS`, `STAGE2_ZONE_WIDTH_PCT` — 착화 단계 난이도
- `STAGE3_TIME_MS`, `STAGE3_SLOTS`(그릴 동시 조리 포인트 수, 기본 5), `STAGE3_COOK_DURATION_MS`,
  `STAGE3_PERFECT_WINDOW`, `STAGE3_MAX_MEATS` — 굽기 단계 난이도. 슬롯 위치는
  `SLOT_POSITIONS` 배열(5개 좌표 정의됨)에서 조정.
- `GRADES` — 등급 판정 점수 구간 및 명칭/아이콘

## 사운드

타이틀/게임방법/명예의 그릴러 화면에서는 `assets/bgm.mp3`(바르고 OTT 드라마 「인생은
고기서 고기까지??」 OST, 원본: `경남도청 홍보콘텐츠 촬영/인생은 고기서 고기까지.mp3`)가
배경음악으로 재생됩니다. "게임 시작"을 누르면 BGM이 멈추고 Web Audio API로 즉석
합성한 효과음(정답/오답 탭, 착화 판정, 굽기 판정 등)으로 전환되며, 결과 화면에
도달하면 BGM이 다시 재생됩니다. 우측 상단 🔊 버튼으로 언제든 음소거 가능(설정은
`localStorage`에 저장). 모바일 브라우저 자동재생 정책상 최초 사용자 탭 이후에
BGM이 시작됩니다.

## 리더보드

`localStorage` 키 `kbbqGameLeaderboard_v1`에 상위 10개 기록 저장 (기기 로컬 저장,
서버 연동 아님). 여러 키오스크 간 랭킹을 공유하려면 경량 백엔드 연동이 필요합니다.

## 알려진 제한사항 (프로토타입 단계)

- **쿠폰 코드는 서버 검증이 없는 클라이언트 생성 코드**입니다. 브라우저 개발자도구로
  결과값을 조작해 임의의 점수/코드를 만들어낼 수 있으므로, 실제 경품 교환 시에는
  ① 부스 스태프가 플레이 전 과정을 육안으로 확인하거나 ② 서버 발급/검증 로직을
  추가하는 것을 권장합니다.
- 리더보드는 기기별 로컬 저장이라 여러 키오스크 간 실시간 공유가 안 됩니다.
- STAGE 1 언박싱 이미지(`assets/cookcam-box.png`)는 적용했지만, 아이템/고기 아이콘은
  아직 이모지입니다. 실제 제품 사진은
  `/Volumes/T7_michael/★바르고★/2025/2025년 경남 디자인주도 제조혁신 지원사업_영문상페/촬영이미지/`
  에 있음 (`패키지누끼.png` — COOKCAM BBQ SOOT 숯 패키지 정컷, `연출컷1~14.jpg` — 실제
  그릴/숯불/고기 연출샷). 정식 에셋으로 교체할 때 `assets/`에 복사해 CSS/HTML의
  이모지를 `<img>`로 바꾸면 됨. 실물 그릴은 사각 스테인리스 석쇠망 + 금테 두른
  검정 하단 트레이 형태이며, 게임 속 STAGE 1 아이템 아이콘(숯/착화탄/석쇠/집게 등)도
  이 패키지 하단의 픽토그램 톤에 맞추면 통일감을 줄 수 있음.

## 네이티브 앱 전환 (선택)

1. **Android**: `~/.bubblewrap` (Bubblewrap CLI)로 이 웹 게임을 TWA로 패키징 —
   `suro-game-twa` 프로젝트 구조 참고.
2. **iOS**: WKWebView 기반 래퍼 프로젝트 생성 — `suro-game-ios` 프로젝트 구조 참고.
3. 두 경우 모두 이 저장소를 정적 호스팅(예: Cloudflare Pages/Netlify)에 올린 뒤
   해당 URL을 래퍼가 로드하도록 설정하면 됩니다.
